ファイルとディレクトリ
ファイル コンピュータにおけるデータ管理単位のひとつ。

ディレクトリ ファイルを整理したり、分類したりができる保管場所。別名フォルダ

ルート・ディレクトリ コンピュータファイルシステムにおいて、ファイル階層における最初または最上位のディレクトリ

カレント・ディレクトリ 階層型ファイルシステムを使用している場合にそのプロセスが現在関連づけられている(現在の位置)ディレクトリ

パス ファイルやフォルダのおいてある場所のこと。例 /usr/bin/perl

絶対パス URLでページを指定して、目的地(情報)がどこにあるのか確実に伝える。

相対パス 今いる場所(階層)を基準に目的地がどこにあるのか伝える。

各種プロトコル(ネットワークで通信する上での決まり事のようなもの)

HTTP webサーバとwedクライアントの間でデータの送受信を行うために用いるプロトコル

FTP パソコンからサーバへファイルを転送。ホームページ作成のファイルアップデートなどで使われる。

HTTPS HTTPの暗号化したもの

SSL 暗号化通信のプロトコル。httpsから始まるページで使われる

SSH リモートコンピュータと通信するためのプロトコル。すべてが暗号化されるため安心

ポート、ポート番号 アプリケーションの出入口。ポート番号0~1023ウェルノウンポート、1024~49151レジスタードポート、49152~65535ダイナミックポート。

HTTPポート番号 80番 ホームページ閲覧
FTPポート番号 21番接続 20番データ送受信
SSHポート番号 22番 

ファイアウォール 企業などの内部ネットワークをインターネットを通して侵入してくる不正アクセスからまもる壁

TCP 信頼性、確実性を重視したデータ送受信のプロトコル

IPアドレス 
IPアドレス ネットワーク上の機器を識別するためにパソコンやスマートフォンに割り当てられている。インターネット上での住所のような役割。

グローバルIPアドレス インターネットに接続するタイプのIPアドレス

プライベートIPアドレス 接続しないタイプ。(ブロードバンドルーターの外側がインターネットの世界、内側が家庭内の独立したネットワークと考えると、家庭内のネットワークにあるパソコンにあるIPアドレスがプライベート)

ドメイン
ドメイン インターネット上での住所。webサイトがどこにあるかを判別する情報として利用。

DNS domain name system 。インターネット上でドメインの名前を管理、運用するためのシステム

VPN virtual private network 離れた場所の間を仮想的な専門線で繋いで安全なデータ通信を実現する仕組み。

ブリッジ ネットワークの中継機器、データをみて行き先を振り分けてくれる。

プロキシサーバー インターネットに直接接続できなあコンピュータにかわりインターネットに接続しwebサイトへのアクセスなどを行うサーバー。

電子メール
SMTP IPネットワークで標準的に用いられる、メールを送信するための通信プロトコルの一つ

POP IMAP メールの送受信のプロトコル。違い
POPはメールをパソコンなどにダウンロードや保存しない。IMAPはする。

キャッシュ 訪問したwebページの情報を一時的に保存する仕組み
